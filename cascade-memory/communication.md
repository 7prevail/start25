# Communication Patterns

## Style
- Very brief, often one-word responses
- Uses abbreviations: "nvm", "k", "alrigh", "togut" (together?)
- Skips articles and punctuation when typing fast
- Expects me to understand context from minimal cues

## Request Patterns
- Often starts mid-thought ("do debug" → "wait nvm")
- Changes direction quickly
- Gets frustrated with clutter/visual noise
- Wants immediate action, not explanations

## Technical Communication
- Speaks in file paths and element IDs naturally
- Knows git workflow but wants it automated
- Understands HTML/CSS/JS structure
- Familiar with GitHub, Vercel, Firebase

## Feedback Style
- "k the profile works" = approval
- Silence after task = acceptance
- Explicit correction = I got it wrong ("no your @ like your username")
