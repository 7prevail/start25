# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

**Contact:** as1730 (GitHub) / 7prevail (alt)

1. **DO NOT** open a public issue
2. Send details via GitHub private vulnerability report or direct message
3. Allow reasonable time for response before public disclosure

## Security Practices

- API keys are stored securely (not in client-side code)
- User data is stored in localStorage (client-side only)
- Firebase rules restrict unauthorized access
- No sensitive data in URL parameters

## Acknowledgments

We appreciate responsible disclosure of security issues.
